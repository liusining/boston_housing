问题 1 - 特征观察¶

如前文所述，本项目中我们关注的是其中三个值:'RM'、'LSTAT' 和'PTRATIO'，对每一个数据点:

  • 'RM' 是该地区中每个房屋的平均房间数量；
  • 'LSTAT' 是指该地区有多少百分比的业主属于是低收入阶层（有工作但收入微薄）；
  • 'PTRATIO' 是该地区的中学和小学里，学生和老师的数目比（学生/老师）。

凭直觉，上述三个特征中对每一个来说，你认为增大该特征的数值，'MEDV'的值会是增大还是减小呢？每一个答案都需要你给出理由。

提示：你预期一个'RM' 值是6的房屋跟'RM' 值是7的房屋相比，价值更高还是更低呢？

问题 1 - 回答：¶

  1. 'RM' 和 'MEDV' 正相关：'RM' 越大的地区，'MEDV' 越大。因为人们可能对房子的使用有很多设想，房间 A 做卧室，B 做书房，C 做活动室。如果房间少，人们对于房子的兴趣相对更低一些，所以价格低。如果房间多，人们的兴趣或许更浓厚，需求更大，价格高。
  2. 'LSTAT' 和 'MEDV' 负相关：'LSTAT' 越大的地区，'MEDV' 越小。因为低收入群体多的地方，人们会普遍克制自己买房的需求，对房子的需求小，房屋价格低。
  3. 'PTRATIO' 和 'MEDV' 负相关：如果生师比（学生/老师）太高，家长可能会担心自己的孩子不能得到老师的关注，因此会影响家长入住此处学区房的需求，房屋价格低。

编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    y_true = np.array(y_true)
    y_predict = np.array(y_predict)
    
    sum_of_true_variance = float(((y_true - y_true.mean()) ** 2).sum())
    sum_of_residual_variance = float(((y_true - y_predict) ** 2).sum())
    
    score = 1 - sum_of_residual_variance / sum_of_true_variance

    return score

什么是网格搜索法？如何用它来优化模型？

问题 7 - 回答:¶

  • 什么是网格搜索法？

网格搜索法（Grid Search）是一种寻找模型最优参数的方法。它是在模型的各种参数取值组合下，比较模型的表现，然后找到模型表现最好的那个参数组合。

它之所以被称为网格，是因为参数的取值组合可以表示为一个网格：由每种参数的取值选项数量相乘得到的网格。

  • 如何用它来优化模型？

使用网格搜索法时，每一种参数组合都会被带入模型。在不同参数的模型被数据训练后，我们会评估所得模型的预测表现，找到表现最好的模型所对应的参数组合。用这个参数组合来配置模型的参数，从而优化模型。

问题 8 - 回答：¶

  • 什么是K折交叉验证法（k-fold cross-validation）？

K折交叉验证法是一种选取训练集和测试集来训练并验证模型的方法。K 折的意思是，将训练用的数据分成 K 份，然后进行 K 次训练：将 K 份数据子集中的第一个的作为测试集，其余的作为训练集，训练并验证模型；然后使用 K 份中的第二个作为测试集，其余的作为训练集…… 依此类推。最后，计算 K 个训练所得模型的预测结果的平均值，作为最后的预测结果。

  • GridSearchCV是如何结合交叉验证来完成对最佳参数组合的选择的？
  1. 在模型的各个参数的取值范围内，把参数的取值组合依次带入模型。
  2. 然后用交叉验证法来分割训练数据，分割成训练集和验证集（validation set)。虽然训练数据已经是从原数据中分割而来的，但在这里，会让训练数据再分割一下，分成训练集和验证集。默认情况下这个切分是按数据的顺序切分的，不过在KFold中使用shuffle=True参数，可以乱序切分。
  3. 在第一个参数组合下，用分割好的训练集来训练模型，然后用剩下的数据，也就是验证集(validation set)来验证模型。如果训练数据被分割成了 K 份，则训练 K 次，依次选取一个训练数据的子集作为验证集，其余的作为训练集。
  4. 计算这个参数组合下，模型对验证集数据所做预测的平均分。
  5. 对其余的参数组合，重复步骤 3 和 4，直至验证了所有的参数组合，并分别计算了训练所得模型对验证集数据所做预测的平均分。
  6. 比较各个参数组合下，训练所得模型对验证集数据预测的平均分，选择平均分最高的那一组的参数组合作为最佳参数组合。
  • GridSearchCV中的'cv_results_'属性能告诉我们什么？

通过在 fit_model 函数中加入 print pd.DataFrame(grid.cv_results_) 语句，以及查看 GridSearchCV 的文档，发现在cv_results_属性会告诉我们以下内容：

  1. mean_fit_time：各个参数条件下，用 K 折交叉验证法训练模型所用的平均时间
  2. mean_score_time：各个参数条件下，对 K 个测试集预测结果进行评分所用的平均时间
  3. mean_test_score：在各个参数组合下，模型根据测试集数据进行的 K 次预测所得平均分
  4. mean_train_score：在各个参数组合下，模型对训练集数据进行的 K 次预测所得平均分
  5. param_XXX：参数XXX的取值范围，如果有多个参数，就会有多个 param_XXX
  6. params：各个的参数组合形成的数组
  7. rank_test_score：各个参数组合下，模型对测试集进行多次预测所得的平均分的排名。
  8. splitN_test_score：原数据集被分为 K 个互斥的子集。在各个参数组合下，当第 N 个数据子集被当做测试集时，模型对测试集的预测所得评分。N 的取值为0 到 K-1。
  9. splitN_train_score：原数据集被分为 K 个互斥的子集。在各个参数组合下，当第 N 个数据子集被当做测试集时，其余的子集被当做训练集，模型对训练集的预测所得评分。N 的取值为0 到 K-1。
  10. std_fit_time：归一化各个参数组合下训练模型的平均时间
  11. std_score_time：归一化各个参数组合下对测试集预测结果评分的平均时间
  12. std_test_score：归一化各个参数组合下模型对于测试集的预测的平均得分
  13. std_train_score：归一化各个参数组合下模型对于训练集的预测的平均得分
  • 网格搜索时如果不使用交叉验证会有什么问题？交叉验证又是如何解决这个问题的？

问题：

如果不在网格搜索中使用交叉验证，我们对一个参数组合下训练得到的模型进行验证时就会面临尴尬：

因为不使用交叉验证，也就没有从原训练集中分割出的验证集（validation set），那么此时我们可以用来验证数据的集合只有训练集（train set）和测试集（test set），

  1. 如果使用训练集，数据都是模型见过的，用它来评估模型无法排除过拟合。这样，有些参数下模型可能评分很高，但却是由于过拟合，会误导我们对最佳参数的选择。
  2. 如果使用测试集，也会有问题：训练集训练得到的模型可能并没有真正的泛化，只是刚好对于测试集过拟合。这样，我们选择最佳参数就面临风险。又因为测试集只有一个，我们也无从验证这个参数组合下的模型是不是真的对测试集过拟合。

总之，都会面临过拟合的风险。

解决：

使用交叉验证，将训练数据再分出一个训练集和验证集（validation set）。这样，在网格搜索的每个参数组合下，模型会根据训练集进行训练，又在验证集（validation set）上验证表现。

利用验证集对不同参数组合下的模型进行评估，我们可以阶段性的确定不同参数组合下的泛化能力，然后选择这一阶段泛化能力最好的参数，作为最优参数。

最后，因为我们到目前为止也并没有使用过测试集（test set），我们还可以用测试集再评估一下最优参数下的模型的表现（这也就是步骤六编程练习5所做的事情）。

from sklearn.model_selection import KFold, GridSearchCV
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer

def fit_model(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    
    cross_validator = KFold(n_splits=10)
    
    regressor = DecisionTreeRegressor(random_state=0)

    params = {'max_depth': range(1, 11)}

    scoring_fnc = make_scorer(performance_metric)

    grid = GridSearchCV(regressor, param_grid=params, scoring=scoring_fnc, cv=cross_validator)

    # 基于输入数据 [X,y]，进行网格搜索
    grid = grid.fit(X, y)

    # 返回网格搜索后的最优模型
    return grid.best_estimator_

问题 13 - 回答：¶

  • 1978年所采集的数据，在已考虑通货膨胀的前提下，在今天是否仍然适用？

不一定。这还取决于其他社会环境，比如政策环境，金融环境等。如果社会环境不一样，即使扣除通货膨胀的部分，人们对于房屋的需求也可能不一样，这反应在价格上，就是价格会更高，或者更低。而如果社会环境基本一致，在扣除通货膨胀的影响后，以前的数据可以适用。

  • 数据中呈现的特征是否足够描述一个房屋？

不足够。根据生活经验，一所房子的周边设施，交通状况等都会影响房屋的价格，而数据集中值记录了低收入群体的比例等3项指标，使得我们无从考察其他的众多与放假的的关系。

  • 在波士顿这样的大都市采集的数据，能否应用在其它乡镇地区？

不一定，这取决于人们在大都市买房时采用的标准，和在乡镇买房时所采用的标准是否一致。如果一致，则从波士顿采集的数据可以应用在乡镇，反之，则不能。

  • 你觉得仅仅凭房屋所在社区的环境来判断房屋价值合理吗？

不合理。根据生活经验，不同的房地产公司，不同的房屋室内布局都可能影响房屋的销售价格。
